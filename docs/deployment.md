# Deployment Guide тАФ Vercel

## Prerequisites
- GitHub / GitLab / Bitbucket repo with this codebase pushed
- Vercel account at vercel.com

## One-time Vercel setup

### 1. Import project
1. Go to vercel.com тЖТ **Add New тЖТ Project**
2. Import your Git repository
3. Vercel auto-detects Next.js тАФ no framework override needed

### 2. Build settings (Vercel auto-fills these)
| Setting | Value |
|---|---|
| Build Command | `npm run build` |
| Output Directory | `.next` (default) |
| Install Command | `npm install` |

> `npm run build` runs `next build && next-sitemap`. Vercel runs this in full, so `sitemap.xml` and `robots.txt` are regenerated on every deploy.

### 3. Environment variables
Set these in **Vercel Dashboard тЖТ Project тЖТ Settings тЖТ Environment Variables**:

| Variable | Example | Required |
|---|---|---|
| `NEXT_PUBLIC_SITE_URL` | `https://urgentprinters.com` | Yes (for sitemap + og URLs) |

> For preview branches, Vercel sets `VERCEL_URL` automatically. The site falls back to `https://urgentprinters.com` if `NEXT_PUBLIC_SITE_URL` is not set.

### 4. Custom domain
1. Vercel Dashboard тЖТ Project тЖТ **Domains**
2. Add `urgentprinters.com` and `www.urgentprinters.com`
3. Update your DNS records as shown by Vercel (A record or CNAME)

## Subsequent deploys
Every `git push` to `main` triggers a production deploy automatically.
Pull request branches get a unique preview URL.

## What the build produces
- **47 routes** тАФ 30 static, 6 SSG (18 product pages), 11 dynamic
- `public/sitemap.xml` тАФ auto-generated by next-sitemap after each build
- `public/robots.txt` тАФ blocks `/account`, `/checkout`, `/cart`, `/auth/`

## Swapping mock data for real APIs
All API functions are in `lib/api/*.ts`. Each function has a `// REAL API:` comment. Replace only the function body тАФ all consumers (pages, hooks) remain unchanged.

| Data | File | Replace with |
|---|---|---|
| Products | `lib/api/products.ts` | Your print management backend |
| Orders | `lib/api/orders.ts` | Order management system |
| Auth | `lib/api/auth.ts` | NextAuth.js / Supabase Auth / Firebase Auth |
| Addresses | `lib/api/addresses.ts` | Backend address CRUD |
| Contact | `app/contact/page.tsx` `onSubmit` | Email service (Resend / SendGrid) |
| Reviews | `app/account/orders/[id]/review/page.tsx` `onSubmit` | Reviews backend |

## Known limitations (pre-production)
- Auth token stored in `localStorage` via Zustand persist тАФ swap for `HttpOnly` cookies in production
- Payment flow is mock тАФ integrate Razorpay / PayU SDK in `app/checkout/page.tsx` `handlePlaceOrder`
- Coupon codes are client-side constants тАФ move to backend validation
- Artwork files are not actually uploaded тАФ wire `ArtworkUpload` to S3 / Cloudflare R2
